ELÇİ YÖNETİM MERKEZİ V3.1 — TEK SEFERLİK KURULUM
================================================

ÖNEMLİ
------
Bu paketi GitHub'ın web ekranından onlarca dosyayı tek tek yükleyerek kurmayın.
Paketle birlikte gelen GUNCELLEMEYI-UYGULA.bat dosyasını ve GitHub Desktop'ı kullanın.

A) GITHUB DESKTOP İLE DEPONUN BİLGİSAYARA ALINMASI
--------------------------------------------------
1. GitHub Desktop'ı açın.
2. File > Clone repository seçin.
3. elciadmin/veteriner-klinik-sitesi deposunu seçin.
4. Clone düğmesine basın.
5. GitHub Desktop'ta Repository > Show in Explorer seçeneğine basın.
6. Açılan klasör yolunu aklınızda tutun.

B) YENİ PAKETİN UYGULANMASI
---------------------------
1. İndirdiğiniz ELCI-YONETIM-MERKEZI-V3-TAM-PAKET.zip dosyasını klasöre çıkarın.
2. Çıkan klasörde GUNCELLEMEYI-UYGULA.bat dosyasını çift tıklayın.
3. İstendiğinde GitHub Desktop'ın açtığı veteriner-klinik-sitesi klasörünü pencereye sürükleyin ve Enter'a basın.
4. Script şu eski ve gereksiz bölümleri kaldırır:
   - content
   - settings
   - eski dist ve node_modules
   - eski content-api fonksiyonu
5. Ardından yeni site dosyalarını doğru yerlere kopyalar.

C) GITHUB'A GÖNDERME
--------------------
1. GitHub Desktop'a dönün.
2. Sol alttaki Summary alanına şunu yazın:
   Yönetim Merkezi V3.1
3. Commit to main düğmesine basın.
4. Üstte Push origin düğmesine basın.
5. Netlify Deploys ekranında yeni yayının Published olmasını bekleyin.

D) NETLIFY'DA BİR KEZ KONTROL EDİLECEK AYARLAR
----------------------------------------------
1. Identity > Registration preferences: Invite only olmalı.
2. Identity > Git Gateway: Enabled/Active olmalı.
   Eski paneliniz çalıştığı için büyük olasılıkla zaten aktiftir.
3. Forms > Usage and configuration bölümünde Form detection açık olmalı.
4. Project configuration > Notifications > Emails and webhooks >
   Form submission notifications bölümüne gidin.
5. online-randevu formu için e-posta bildirimi ekleyin:
   elcivetklinik@gmail.com
6. İsteğe bağlı olarak Environment variables bölümüne şunu ekleyin:
   APPOINTMENT_RETENTION_DAYS = 180
   Bu ayar randevu kayıtlarını 180 gün sonra otomatik temizler.

GİZLİ ANAHTAR GEREKMİYOR
------------------------
Bu sürüm GitHub kişisel erişim anahtarı istemez. Mevcut Netlify Identity ve
Git Gateway bağlantısını kullanır.

E) İLK KONTROL
--------------
1. /admin/ adresine giriş yapın.
2. Blog bölümünde geçici bir TASLAK oluşturun.
3. Taslağı açıp düzenleyin, sonra arşive kaldırın ve geri getirin.
4. Sitede 6 ana sayfa SSS'sini, 6 Google yorumunu ve Instagram film şeridini kontrol edin.
5. Randevu formundan kendi bilgilerinizle bir test talebi gönderin.
6. Talebin:
   - Netlify Forms'a,
   - yönetim panelindeki Randevular bölümüne,
   - elcivetklinik@gmail.com adresine
   geldiğini doğrulayın.

Değişikliklerin sitede görünmesi GitHub kaydı ve Netlify deploy nedeniyle genellikle
Netlify deploy tamamlandığında görünür.
